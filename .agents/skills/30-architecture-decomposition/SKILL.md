# Architecture Decomposition

## 目的

将产品计划拆解为技术架构、模块、接口、数据流、部署、安全、可观测性和演进策略。

## 必做

- 基于 GitHub repo、本地 repo、docs 和官方资料。
- 识别已有架构与新增架构的边界。
- 将架构模块映射到 Milestones 和 Issues。
- 为高风险技术决策输出 ADR 草案。

## 输出

- 架构概览。
- 模块分解。
- 数据模型/API 边界。
- 依赖关系图。
- ADR 草案。
- 对 Linear Issue 拆分的影响。
