# /extend-project

目标：基于现有 Linear Project 和事实来源，为新增需求做影响分析与追加编排。

必须先读取：
- Linear Project、Milestones、Issues、Project Updates。
- 关联 GitHub repo 的 README、package、最近 PR/commits。
- 本地 repo/docs 中与需求相关的文件。
- 必要时联网查官方资料。

输出：
- 当前基线。
- 新需求事实与假设。
- 影响面：产品、架构、数据、API、任务、周期、风险。
- 新增/修改/不建议修改的 Milestones 和 Issues。
- dry-run 写入计划。
